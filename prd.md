# 여름행사 등록 PRD

## 0. 문서 개요

- 문서 목적: 교회 연합 여름행사 등록 웹앱 개발을 위한 제품 요구사항 정의
- 앱 이름: 여름행사 등록
- 앱 형태: 모바일 우선 반응형 웹앱
- 주요 대상: 학부모, 교사/봉사자, 교회 담당자, 본부/지방/연합 관리자
- 개발 목표: 등록, 교회별 관리, 참가비 관리, 엑셀 다운로드, 같은 조 요청, 자동 조편성까지 포함한 완성형 1차 출시

# 1. PRD 문서

## 1-1. 프로젝트 개요

### 웹앱 이름

여름행사 등록

### 한 줄 설명

교회 연합 여름행사의 참가 등록, 교회별 관리, 참가비 확인, 엑셀 다운로드, 자동 조편성까지 처리하는 웹앱

### 앱 유형

교회 연합 행사 등록 및 관리 웹앱

### 예상 난이도

중상

### 난이도 판단 이유

- 학부모는 로그인 없이 등록해야 함
- 교회 담당자와 본부 관리자는 로그인 권한이 필요함
- 행사별 설정값을 따로 관리해야 함
- 참가 유형별, 교회별 참가비 계산이 필요함
- 등록 데이터를 다양한 기준으로 필터링하고 엑셀 다운로드해야 함
- 같은 조 요청과 자동 조편성 기능이 포함됨
- 자동 조편성 후 수동 수정과 최종 확정 기능이 필요함

### 핵심 목적

교회 연합 여름행사 등록 업무를 온라인으로 통합하여, 학부모는 쉽게 등록하고, 교회 담당자는 자기 교회 등록 현황을 관리하며, 본부/지방/연합 관리자는 전체 등록 현황, 참가비, 조편성, 엑셀 자료를 효율적으로 관리할 수 있도록 한다.

## 1-2. 사용자 시나리오

## 사용자 유형

### 1. 학부모 / 개별 등록자

- 행사 등록 링크를 통해 접속한다.
- 별도 회원가입 없이 자녀 정보를 등록한다.
- 등록 후 수정이 필요하면 이름, 연락처, 수정용 비밀번호로 등록 내용을 수정한다.
- 등록 마감 후에는 직접 수정할 수 없으며, 소속 교회 담당자에게 문의한다.
- 학부모 화면에서는 참가비, 계좌번호, 납부 상태를 보지 않는다.

### 2. 교사 / 봉사자

- 본인이 직접 등록할 수 있다.
- 교회 담당자가 대신 등록할 수도 있다.
- 본부 관리자가 직접 등록할 수도 있다.
- 보호자 정보 대신 본인 연락처와 역할을 입력한다.

### 3. 교회 담당자

- 로그인 후 자기 교회 등록자만 확인하고 관리한다.
- 자기 교회 등록자를 추가하거나 수정할 수 있다.
- 자기 교회 등록자의 같은 조 요청을 입력할 수 있다.
- 자기 교회의 등록 인원, 참가비 총액, 본부 계좌, 납부 상태를 확인한다.
- 납부 상태는 볼 수만 있고 수정할 수 없다.

### 4. 본부 / 지방 / 연합 관리자

- 전체 행사를 생성하고 관리한다.
- 참여 교회와 교회 담당자를 관리한다.
- 전체 등록자를 조회, 수정, 필터링한다.
- 참가비 설정과 납부 상태를 관리한다.
- 엑셀 다운로드를 실행한다.
- 같은 조 요청을 확인한다.
- 자동 조편성을 실행하고 수동으로 수정한 뒤 최종 확정한다.

## H. 교회 담당자 계정 생성 및 승인

교회 담당자 계정은 두 가지 방식으로 생성할 수 있다.

### 방식 1. 본부 관리자가 직접 생성

본부 관리자가 교회와 교회 담당자 정보를 직접 입력한다.

입력 항목:

- 교회명
- 담당자 이름
- 담당자 연락처
- 로그인 아이디
- 임시 비밀번호
- 메모

### 방식 2. 교회 담당자가 가입 신청

교회 담당자가 직접 가입 신청을 하고, 본부 관리자가 승인 또는 반려한다.

가입 신청 항목:

- 교회명
- 담당자 이름
- 담당자 연락처
- 로그인 아이디
- 비밀번호
- 비고

### 승인 후 처리

- 승인된 사용자는 교회 담당자 권한을 가진다.
- 해당 교회의 등록자만 조회하고 수정할 수 있다.

## I. 참가비 관리

개별 학부모가 직접 입금하지 않고, 각 교회가 등록금을 모아 대표로 본부 계좌에 입금한다.

### 학부모 화면

- 참가비 정보 표시하지 않음
- 계좌번호 표시하지 않음
- 이체 안내 문구 표시하지 않음

### 교회 담당자 화면

교회 담당자에게만 본부 계좌와 총 납부 금액을 보여준다.

표시 정보:

- 총 등록 인원
- 참가 유형별 인원
- 참가 유형별 금액
- 총 납부 금액
- 본부 입금 계좌
- 예금주
- 계좌번호 복사 버튼
- 납부 상태
- 납부 확인 지연 안내 문구

### 참가비 설정 방식

본부 관리자는 참가 유형별로 참가비를 설정할 수 있다.

예시 1:

| 참가 유형 | 금액 |
|---|---:|
| 학생 | 20,000원 |
| 교사 | 0원 |
| 봉사자 | 0원 |

예시 2:

| 참가 유형 | 금액 |
|---|---:|
| 유아부 | 10,000원 |
| 유치부 | 10,000원 |
| 초등부 | 20,000원 |
| 중등부 | 25,000원 |
| 교사 | 15,000원 |

### 교회별 참가비 예외

교회별로 참가비가 다를 수 있으므로, 본부 관리자가 교회별 참가비 예외를 설정할 수 있어야 한다.

예시:

- A교회 학생 참가비: 20,000원
- B교회 학생 참가비: 15,000원
- C교회 교사 참가비: 무료

### 납부 상태

교회별 납부 상태는 본부 관리자가 변경한다.

상태값:

- 미납
- 확인 필요
- 납부완료

### 권한

| 사용자 | 납부 상태 조회 | 납부 상태 수정 |
|---|---:|---:|
| 학부모 | 불가 | 불가 |
| 교회 담당자 | 가능 | 불가 |
| 본부 관리자 | 가능 | 가능 |

## J. 계좌번호 복사 기능

### 기능 요구사항

## M. 자동 조편성 기능

자동 조편성은 1차 출시 범위에 포함한다.

단, 자동 조편성은 최종 확정 기능이 아니라 **조편성 초안 생성 기능**으로 제공한다. 본부 관리자가 자동 생성 결과를 확인하고 수동 수정한 뒤 최종 확정한다.

### 조편성 방식

본부 관리자가 조편성 그룹을 직접 만든다.

예시:

| 조편성 그룹 | 포함 대상 |
|---|---|
| 유치부 | 유아부, 유치부 |
| 초등 저학년 | 초등 1학년, 초등 2학년, 초등 3학년 |
| 초등 고학년 | 초등 4학년, 초등 5학년, 초등 6학년 |
| 중등부 | 중등부 |

### 조편성 그룹 설정값

- 조편성 그룹명
- 포함할 부서/학년
- 조 개수
- 조당 인원 수
- 성별 균형 적용 여부
- 교회 분산 적용 여부
- 같은 조 요청 반영 여부
- 참석 일정 반영 여부
- 교사/봉사자 배정 여부

### 자동 조편성 고려 기준

- 부서/학년
- 성별 균형
- 교회 분산
- 같은 조 요청
- 참석 일정
- 교사/봉사자 배정
- 조별 인원 수 균형

### 자동 조편성 흐름

1. 본부 관리자가 조편성 그룹을 생성한다.
2. 포함할 부서/학년을 선택한다.
3. 조 개수 또는 조당 인원 수를 입력한다.
4. 자동 조편성을 실행한다.
5. 조별 명단 미리보기를 확인한다.
6. 본부 관리자가 필요한 경우 수동으로 수정한다.
7. 최종 확정한다.
8. 조별 명단을 엑셀로 다운로드한다.

### 수동 수정 기능

본부 관리자는 자동 조편성 결과를 수동으로 수정할 수 있어야 한다.

수동 수정 방식:

- 등록자를 다른 조로 이동
- 특정 등록자를 미배정 상태로 변경
- 미배정 등록자를 특정 조에 배정
- 같은 조 요청 반영 상태 변경
- 조 이름 수정
- 조 삭제
- 조 추가

- 이름
- 성별
- 참가 유형
- 부서/학년
- 출생년도
- 소속 교회

- 사진 촬영 동의 여부
- 참석 일정
- 수정용 비밀번호
- 비고

- 참가자 기본 정보
- 소속 교회
- 행사 ID
- 참가 유형
- 참석 일정

- 등록 일시
- 수정 일시

### 결과 화면

- 등록 완료 안내
- 등록자 이름
- 소속 교회
- 참석 일정
- 수정 안내 문구

## 1-8. 제약 사항

## 예산

- 별도 확정 필요
- MVP 이상 범위이므로 단순 등록 앱보다 개발 비용이 높을 수 있음

## 개발 기간

- 자동 조편성까지 포함한 완성형 1차 출시를 목표로 함
- 실제 기간은 개발 인원과 디자인 수준에 따라 달라짐

## 기술 난이도

- 중상
- 권한 관리, 엑셀 다운로드, 조편성 알고리즘, 행사별 설정 구조가 포함됨

## 운영 인원

- 최소 본부 관리자 1명 필요
- 각 교회별 교회 담당자 1명 이상 필요

## 로그인 필요 여부

| 사용자 | 로그인 |
|---|---|
| 학부모 / 개별 등록자 | 불필요 |
| 교사 / 봉사자 직접 등록 | 불필요 |
| 교회 담당자 | 필요 |
| 본부 관리자 | 필요 |

## 관리자 필요 여부

필요함.

관리자 유형:

- 본부 / 지방 / 연합 관리자
- 교회 담당자

## 개인정보 고려사항

수집되는 개인정보:

- 이름
- 성별
- 출생년도
- 보호자 이름
- 보호자 연락처
- 본인 연락처
- 알레르기/건강상 주의사항
- 사진 촬영 동의 여부

주의사항:

- 개인정보는 행사 운영 목적에만 사용해야 한다.
- 행사 종료 후 개인정보 보관 기간을 정해야 한다.
- 엑셀 다운로드 권한은 본부 관리자 중심으로 제한해야 한다.
- 보호자 연락처와 건강 정보는 민감하게 관리해야 한다.
- 수정용 비밀번호는 원문 저장하지 않고 해시 처리해야 한다.

# 3. 화면별 와이어프레임 설명

## 3-1. 학부모 등록 시작 화면

### 화면의 목적

학부모가 행사 정보를 확인하고 등록을 시작하도록 한다.

### 주요 구성 요소

- 행사명
- 행사 설명
- 행사 기간
- 등록 가능 상태
- 등록하기 버튼
- 등록 수정하기 버튼

### 사용자 행동

- 등록하기 버튼을 누른다.
- 등록 수정하기 버튼을 눌러 기존 등록 정보를 수정한다.

### 다음 화면 이동

- 등록하기 버튼 클릭 → 참가자 등록 화면
- 등록 수정하기 버튼 클릭 → 등록 수정 인증 화면

---

## 3-2. 참가자 등록 화면

### 화면의 목적

참가자의 기본 정보를 입력한다.

### 주요 구성 요소

- 참가 유형 선택
- 소속 교회 선택
- 이름 입력
- 성별 선택
- 부서/학년 버튼 선택
- 출생년도 선택
- 보호자 정보 입력
- 티셔츠 사이즈 선택
- 건강상 주의사항 입력
- 사진 촬영 동의
- 참석 일정 선택
- 수정용 비밀번호 입력
- 비고 입력
- 등록하기 버튼

### 사용자 행동

- 필요한 정보를 입력한다.
- 등록하기 버튼을 누른다.

### 다음 화면 이동

- 등록 성공 → 등록 완료 화면
- 입력 오류 → 현재 화면에서 오류 메시지 표시

## 3-3. 등록 완료 화면

- 필요하면 추후 등록 수정 화면으로 이동한다.

### 다음 화면 이동

- 홈으로 이동

- 인증 정보를 입력한다.

### 다음 화면 이동

- 인증 성공 → 등록 수정 화면

# 5. 개발자 또는 AI 코딩 도구에게 전달할 최종 요청문

Next.js, Supabase, Vercel을 사용하여 “여름행사 등록” 웹앱을 개발해주세요.

이 앱은 교회 연합 여름행사의 참가 등록과 관리를 위한 웹앱입니다.  
사용자 권한은 학부모/개별 등록자, 교회 담당자, 본부/지방/연합 관리자로 나뉩니다.

학부모는 로그인 없이 등록 링크로 접속해 자녀 정보를 등록합니다. 등록 항목은 이름, 성별, 참가 유형, 부서/학년, 출생년도, 소속 교회, 보호자 이름, 보호자 연락처, 티셔츠 사이즈, 알레르기/건강상 주의사항, 사진 촬영 동의 여부, 참석 일정, 수정용 비밀번호, 비고입니다. 학부모 화면에는 참가비나 계좌번호를 보여주지 않습니다.

교사와 봉사자도 등록 대상입니다. 교사/봉사자는 본인이 직접 등록하거나 교회 담당자가 대신 등록할 수 있습니다. 교사/봉사자는 보호자 정보 대신 본인 연락처와 역할을 입력합니다.

학부모 또는 개별 등록자는 회원가입 없이 이름, 연락처, 수정용 비밀번호로 등록 내용을 수정할 수 있어야 합니다. 단, 본부 관리자가 설정한 수정 가능 기간이 지나면 직접 수정할 수 없습니다. 수정 가능 기간이 지나면 “수정이 필요한 경우 소속 교회 담당자에게 문의해주세요.”라는 안내 문구를 보여주세요.

교회 담당자는 로그인 후 자기 교회의 등록자만 확인하고 수정할 수 있습니다. 또한 자기 교회 등록자의 같은 조 편성 요청을 입력할 수 있습니다. 교회 담당자는 자기 교회의 총 등록 인원, 참가비 총액, 본부 입금 계좌, 계좌번호 복사 버튼, 납부 상태를 확인할 수 있습니다. 납부 상태는 볼 수만 있고 수정할 수 없습니다. 납부 상태 영역에는 “입금 확인은 본부/지방/연합 관리자가 직접 처리합니다. 확인이 조금 늦어질 수 있습니다.” 문구를 표시해주세요.

본부 관리자는 전체 행사를 생성하고 관리합니다. 하나의 앱에서 여러 행사를 만들 수 있어야 합니다. 행사별로 행사명, 행사 기간, 등록 기간, 수정 가능 기간, 부서/학년 선택지, 출생년도 선택지, 티셔츠 사이즈 선택지, 참가 유형별 참가비, 교회별 참가비 예외, 본부 입금 계좌, 참여 교회, 교회 담당자 계정을 설정할 수 있어야 합니다.

교회 담당자 계정은 본부 관리자가 직접 만들 수도 있고, 교회 담당자가 가입 신청을 하면 본부 관리자가 승인 또는 반려할 수도 있어야 합니다.

본부 관리자는 전체 등록자 목록을 교회별, 부서/학년별, 참가 유형별, 성별, 티셔츠 사이즈별, 참석 일정별, 납부 상태별로 필터링할 수 있어야 합니다. 엑셀 다운로드 시 출력할 항목을 체크박스로 선택할 수 있어야 합니다. 선택 가능한 항목은 이름, 참가 유형, 성별, 부서/학년, 출생년도, 소속 교회, 보호자 이름, 보호자 연락처, 티셔츠 사이즈, 알레르기/건강상 주의사항, 사진 촬영 동의 여부, 참석 일정, 같은 조 요청, 배정된 조, 비고입니다.

엑셀 다운로드 프리셋으로 전체 등록자 명단, 교회별 등록자 명단, 부서/학년별 명단, 교회별 티셔츠 사이즈 요약, 교회별 티셔츠 사이즈 상세, 알레르기/건강 주의사항 명단, 사진 촬영 미동의자 명단, 조별 명단, 납부 상태 현황을 제공해주세요.

자동 조편성 기능을 구현해주세요. 본부 관리자가 조편성 그룹을 직접 만들고, 각 그룹에 포함할 부서/학년을 선택할 수 있어야 합니다. 조 개수 또는 조당 인원 수를 입력하면 자동으로 조편성 초안을 생성합니다. 자동 조편성은 부서/학년, 성별 균형, 교회 분산, 같은 조 요청, 참석 일정, 교사/봉사자 배정, 조별 인원 균형을 고려해야 합니다. 자동 조편성 후 본부 관리자가 수동으로 수정하고 최종 확정할 수 있어야 합니다. 조별 명단도 엑셀로 다운로드할 수 있어야 합니다.

학부모 화면은 모바일 우선, 밝고 따뜻한 파스텔톤 디자인으로 만들어주세요. 교회 담당자와 본부 관리자 화면은 표, 검색, 필터, 상태 표시 중심의 실무적인 관리자 UI로 만들어주세요.

개인정보 보호를 위해 수정용 비밀번호는 원문으로 저장하지 말고 해시 처리해주세요. 학부모는 본인 등록 수정 외의 데이터에 접근할 수 없어야 하며, 교회 담당자는 자기 교회 데이터만 접근할 수 있어야 합니다. 본부 관리자만 전체 데이터와 엑셀 다운로드 기능에 접근할 수 있도록 권한을 분리해주세요.
